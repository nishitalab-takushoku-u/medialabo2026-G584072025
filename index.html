各授業回の学習事項

第1回 WWW と HTML

(今回学習した事柄を説明してください)(100文字以上)

第1回の授業では、HTMLの基本構造について学習しました。ブラウザがどのようにWebページを表示するのかを理解し、h1〜h6タグ、pタグ、imgタグなどの基本的な要素を使って文書を作成する方法を習得しました。自分で書いたコードがブラウザに表示されたときは、ウェブ制作の第一歩を踏み出せた気がして楽しかったです。

演習の成果： 演習1-4:HTMLのさまざまな要素

第2回 CSS による Webページの装飾

第2回の授業では、CSSを使ってウェブページをデザインする方法を学びました。文字の色や大きさを変えたり、背景に色をつけたりする方法を習得しました。HTMLだけで書いたシンプルな文章が、CSSを使うことで一気にきれいで見やすくなることが分かり、デザインの楽しさを実感しました。

演習の成果： ウェブブラウザの紹介

第3回 JavaScript の基本

第3回の授業では、プログラミングのJavaScriptについて学びました。データを保存する「変数」や、条件によって動きを変える「if文」などの基本を習得しました。今までのHTMLやCSSとは違って、頭を使って計算や仕組みを考えるのが難しかったですが、動くページを作るための大切な基礎を学べました。

演習の成果： JavaScript のオブジェクト

第4回 DOM と JavaScript

第4回の授業では、JavaScriptを使ってHTMLの文字や色を自由に書き換える方法（DOM操作）を学びました。ボタンをクリックしたときに画面が変化するような、使う人とやり取りができるページの作り方が分かりました。これでウェブサイトに色々な仕掛けを作れるようになりました。

演習の成果： 足し算ページ

第5回 イベント処理

今日学習したのは、JavaScriptを使ってウェブ画面にデータを表示する方法です。今まではデータをコンソールにだけ出力していましたが、今回はコードで新しいdiv⁠要素を作り、そこに天気などのデータを組み込んで、ページの最後に追加する処理を学びました。HTMLとJSを連携させる良い練習になりました。

演習の成果： DOM の演習その2

工夫点,苦心点,強調点,感想

1.工夫点

このウェブサイト制作において、私が一番工夫したところは、ユーザーの使いやすさと見た目の「シンプルで高級感のあるデザイン」のバランスです。 まず、背景には自分で用意した世界地図の画像（⁠worldmap.png⁠）を HTML に配置し、CSS の ⁠filter: brightness⁠ を使って少し暗めの絶妙なトーンに調整しました。これにより、上に表示される文字や検索ボックスがとても見やすくなりました。 さらに、検索ボックスや天気結果のカードには、授業で習った「半透明の背景色（⁠rgba⁠）」に「丸み（⁠border-radius⁠）」と「ボックスの影（⁠box-shadow⁠）」を組み合わせました。背景の世界地図がうっすらと綺麗に透けて見えるため、ただの四角い箱を表示するよりも、立体的で非常におしゃれなデザインに仕上げることができました。 また、一番の工夫点として、都市名を入力する際に、大文字・小文字のミスや、日本語・英語・中国語のどれで入力しても正しく天気が取得できるようにしました。これは授業では習っていない応用的な内容だったため、天気 API の公式ドキュメント（参考文献）を自分で調べ、各都市の固有のID（⁠City ID⁠）を JavaScript の ⁠if...else if⁠ 文で紐付けることで実現しました。ユーザーがどの言語で入力してもストレスなくエラーなしで検索できる、実用的なシステムに仕上げました。

2.苦心点

一番苦労したところは、背景の色の調整と、JavaScript のエラーの解決です。 最初は背景にネイビーのグラデーションを重ねていたのですが、自分で用意した世界地図の色のせいで、画面全体が「紫色」っぽくなってしまい、デザイン的に納得がいきませんでした。不自然に白くなったり紫になったりするのを防ぐために、CSS のコードを何度も書き直し、最終的に HTML に直接画像を入れ、CSS で明るさをコントロールする形にたどり着きました。 また、JavaScript の開発中に「⁠Uncaught TypeError: can't access property "className", resultBox is null⁠」というエラーが出てしまいました。最初は「東京」の検索時に専用のカード背景画像を表示させようと挑戦していたのですが、コードの実行タイミングのせいで、まだ画面に存在しない要素（⁠null⁠）を触ろうとしていたことが原因だと気づくまでに、とても時間がかかりました。 最終的に、すべての都市で统一された美しいデザインを維持する方がシンプルで完成度が高くなると判断し、東京の画像の実装は諦めました。しかし、このエラーを解決するために、データを読み込み終わった後に処理を行うという順番を並び替えるデバッグ作業を行ったことは、非常に良い勉強になりました。

3.強調点

この作品の強調したいポイントは、余計な装飾を削ぎ落としたことで生まれた「画面の安定感と見やすさ」です。 色々試した結果、アニメーションなどの複雑な動きを入れない方が、かえって背景の世界地図や半透明のカードがきれいに引き立つことに気づきました。また、配置には学校で習った基本の ⁠text-align: center⁠ や ⁠margin: 0 auto⁠ だけを使っているので、どんな画面サイズのパソコンで見てもレイアウトが絶対に崩れません。シンプルですが、誰が見てもパッと天気が頭に入ってくるような、実用的なページに仕上げたところが一番のアピールポイントです。

4.感想

最初は、難しい応用コードをたくさん使えば良いページが作れると思っていましたが、エラーばかりで上手くいきませんでした。しかし、習った技術である「影」や「透明度」のバランスを丁寧に組み合わせるだけで、十分にプロっぽくて高級感のあるページが作れると分かり、とても感動しました。 途中で画像が表示されなかったり、エラーが出たりしたときは諦めそうになりましたが、一つずつ原因を調べて解決していく中で、HTML、CSS、JavaScript の繋がりが深く理解できました。最終的にシンプルで美しい天気アプリが完成して、とても大きな達成感を得ることができました。
